Sliding Window · Family B · Prefix Sum + Monotonic Deque

Why not two-pointer? With negatives, removing the left element can increase the sum — so "shrink while valid" breaks. Instead: build prefix sums P[], then for each R find the largest L where P[R]−P[L] ≥ X and R−L is minimised. A monotonic deque (increasing prefix sums) makes this O(n).

// build prefix sums
vector<int> P(n+1,0);
for(int i=0;i<n;i++) P[i+1]=P[i]+a[i];
deque<int> dq; int ans=INT_MAX;
for(int R=0;R<=n;R++){
  while(!dq.empty()&&
    P[R]-P[dq.front()]>=x){
    ans=min(ans,R-dq.front());
    dq.pop_front();
  }
  while(!dq.empty()&&
    P[dq.back()]>=P[R])
    dq.pop_back();
  dq.push_back(R);
} return ans; }
