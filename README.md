## What is GBMvisuAL?

Most algorithm resources show you the code. GBMvisuAL shows you the **thought process**.

Every visualizer is a step-by-step, interactive animation of an algorithm executing ‚ÄĒ not a static diagram, not a video, not pseudocode. You control the speed, you pause at any step, you watch the data structures mutate in real time as the algorithm makes each decision.

> *"I understood binary search in 10 minutes after watching it. Not in 10 hours of reading about it."*

## ūüéĮ What makes each visualizer different

Every visualizer in GBMvisuAL is built around **one core principle**: show the *decision*, not just the *result*.

For every step of every algorithm, you see:

- **What the algorithm is looking at** ‚ÄĒ highlighted in the data structure
- **What decision it's making** ‚ÄĒ the branching condition, the comparison, the pick
- **Why it made that decision** ‚ÄĒ the current state that caused it
- **What changed** ‚ÄĒ the mutation, the update, the push/pop/merge

This is not a textbook rendering engine. Each visualizer is hand-crafted for its specific algorithm ‚ÄĒ the bar chart for maximum subarray is different from the edge-relaxation panel for Dijkstra's, which is different from the Trie node expansion for grid word search.

## ūü§Ě Contributing

Contributions are what make this library grow. Here's how to add a new visualizer:

### Adding a new visualizer

**1. Create your HTML file** in the appropriate category folder:

```
02-Sliding-Window/11_my_new_visualizer.html
```

**2. Follow the visualizer contract:**

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>GBMvisuAL ‚ÄĒ [Algorithm Name]</title>
  <!-- Self-contained: all CSS and JS in one file -->
</head>
<body>
  <!-- Required elements -->
  <div id="controls">          <!-- Step / Play / Reset / Speed -->
  <div id="visualization">     <!-- The main animation canvas -->
  <div id="state-panel">       <!-- Current state of all data structures -->
  <div id="explanation">       <!-- What is happening THIS step and why -->
</body>
</html>
```

**3. Visualizer quality bar** ‚ÄĒ every visualizer must show:
- [ ] Step-by-step control (not just play/pause ‚ÄĒ individual step forward/back)
- [ ] Current state of all relevant data structures at every frame
- [ ] The *decision* being made at each step (not just the action)
- [ ] Color-coded highlights that correspond to the algorithm's logic
- [ ] At least 3 example inputs the user can switch between
- [ ] A complexity badge showing time and space

**4. Update `index.html`:**
- Find your category section
- Change `card soon` to `card built` on your card
- Add `href="your/path.html"` to the anchor
- Update the live count in the `cat-badge` span
- Update `statBuilt` auto-counts (it's automatic via JS)

**5. Submit a PR** with title format: `[Category] Add [Algorithm Name] visualizer`

### Good first issues

Look for issues tagged [`good first issue`](https://github.com/yourusername/gbmvisual/issues?q=is%3Aopen+label%3A%22good+first+issue%22) ‚ÄĒ these are visualizers that are well-specified and straightforward to build.
